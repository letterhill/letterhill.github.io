파이썬 래퍼 XGBoost 하이퍼 파라미터

  • 사이킷런 래퍼/파이썬 래퍼 XGBoost의 파라미터에는 차이점이 있음 ## 일반 파라미터
  • 디폴트 값을 바꾸는 경우 거의 X
  • booster: gbtree(default), gblinear
  • silent: 0(default), 1(출력 메시지X)
  • nthread: CPU의 실행 스레드 개수 조정 ## 부스터 파라미터
  • 트리 최적화, 부스팅, regularization 등 관련
  • eta [default=0.3, alias: learning_rate]: 학습률 파라미터, 보통 0.01~0.2
  • num_boost_rounds: n_estimators
  • min_child_weight[default=1]: 과적합 조절, 클수록 분할 자제
  • gamma[default=0, alias: min_split_loss]: 해당 값보다 큰 손실이 감소된 경우 리프 노드 분리, 클수록 과적합 감소
  • max_depth[default=6]: 보통 3~10
  • sub_samplt[default=1]: 0.5~1 ## 학습 태스크 파라미터 : 학습 수행 시의 객체 함수, 평가를 위한 지표 등

과적합이 심각하다면?

  • eta 값 낮추기
  • max_depth 낮추기
  • min_child_weight, gamma 값 높이기
  • subsample, colsamplt_bytree 조정

Early Stopping (조기 중단) 기능

: n_estimators에 도달하지 않더라도 예측 오류가 더 이상 개선되지 않으면 반복을 끝까지 수행X

# DMatrix: 파이썬 래퍼 XGBoost 만의 전용 데이터 객체

params = {'max_depth':3,

Code

from sklearn.metrics import confusion_matrix, accuracy_score
from sklearn.metrics import precision_score, recall_score
from sklearn.metrics import f1_score, roc_auc_score

def get_clf_eval(y_test, pred=None, pred_proba=None):
    confusion = confusion_matrix( y_test, pred)
    accuracy = accuracy_score(y_test , pred)
    precision = precision_score(y_test , pred)
    recall = recall_score(y_test , pred)
    f1 = f1_score(y_test,pred)
    # ROC-AUC 추가 
    roc_auc = roc_auc_score(y_test, pred_proba)
    print('오차 행렬')
    print(confusion)
    # ROC-AUC print 추가
    print('정확도: {0:.4f}, 정밀도: {1:.4f}, 재현율: {2:.4f},\
    F1: {3:.4f}, AUC:{4:.4f}'.format(accuracy, precision, recall, f1, roc_auc))
